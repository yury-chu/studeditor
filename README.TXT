Приложение по тестовому заданию.

Среда разработки:
Ubuntu 14.04 LTS
Google Chrome
PyCharm 4.5.3

Python 2.7.6
Django 1.8.3
Разработка ведется на Sqlite3.

Необходимые модули прописаны в requirements.txt
(Содержит модуль необходимый для перевода проекта на Postgresql)

На данный момент не реализован поиск, и сортировка по имени и т.п.
Не валидные html страницы, нет разбивок на блоки, расширения.
Нету визуального оформления с помощью статики.

Реализовано:
-Выбор роли пользователя
-Добавление групп (только для администратора)
-Добавление студента.
-Отображение списка студентов: 
	для администратора все записи,
	для документовода только те, что он добавил сам.

Разрабатывал с пониманием того, что фреймворк django знаю не очень уверенно.
Поэтому не использованы оптимальные для данных задач модули.
В связи с чем пока сделал как смог, на основе имеющихся знаний.
Приложение очень уязвимо и открыто. 
Проверок авторизации нету. Можно в дальнейшем реализовать через встроенную систему авторизации django.

Работу с админкой не вел. (По условиям из ТЗ)
